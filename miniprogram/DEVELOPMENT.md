# 基民眼微信小程序 - 开发说明

## 从 Next.js 迁移到微信小程序的改造说明

### 主要改动

1. **框架迁移**
   - 从 Next.js (React) 改为微信小程序原生框架
   - 页面结构从 JSX 改为 WXML
   - 样式从 CSS 改为 WXSS
   - 数据管理从 React Hooks 改为小程序 Page 生命周期

2. **登录系统**
   - 原项目使用 Supabase 邮箱登录
   - 小程序版本使用微信登录 (wx.login + wx.getUserProfile)
   - 支持本地模式和云开发模式

3. **数据接口**
   - 原项目使用浏览器 JSONP 和 Script 标签注入
   - 小程序版本使用 wx.request 并手动解析 JSONP
   - 需要配置合法域名白名单

4. **数据存储**
   - 原项目使用 localStorage 和 Supabase
   - 小程序版本使用 wx.storage (本地) + 云开发（可选）

5. **UI设计**
   - 保留原项目的玻璃拟态设计风格
   - 适配小程序组件和交互规范
   - 优化移动端触摸体验

### API改造细节

#### 原始实现（浏览器环境）

```javascript
// 使用 Script 标签注入
const script = document.createElement('script');
script.src = url;
document.body.appendChild(script);
```

#### 小程序实现

```javascript
// 使用 wx.request
wx.request({
  url: url,
  dataType: 'other',
  success: (res) => {
    // 手动解析 JSONP: jsonpgz({...})
    const json = res.data.replace(/^jsonpgz\(/, '').replace(/\);?$/, '');
    const data = JSON.parse(json);
  }
});
```

### 开发注意事项

1. **网络请求**
   - 必须使用 HTTPS
   - 必须在公众平台配置合法域名
   - 本地开发可以不校验域名

2. **跨域问题**
   - 小程序不存在浏览器跨域限制
   - 但需要配置服务器域名白名单

3. **数据大小限制**
   - 单次 setStorage 最大 10MB
   - 总存储空间 10MB
   - 建议定期清理旧数据

4. **性能优化**
   - 避免频繁 setData
   - 合理使用 wx:if 和 hidden
   - 图片使用 webp 格式

5. **兼容性**
   - 基础库版本要求 >= 2.10.0
   - 支持 iOS 和 Android
   - 注意不同版本的 API 差异

## 常见问题

### 1. 网络请求失败

**原因**: 未配置合法域名  
**解决**: 在微信公众平台配置服务器域名

### 2. 登录失败

**原因**: AppID 配置错误  
**解决**: 检查 project.config.json 中的 appid

### 3. 数据不刷新

**原因**: 缓存问题  
**解决**: 清除缓存或重新编译

### 4. 真机预览白屏

**原因**: 代码错误或网络问题  
**解决**: 查看真机调试控制台
