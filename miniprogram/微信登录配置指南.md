# 微信登录配置指南

本项目支持三种微信登录模式，您可以根据自己的需求选择：

## 📋 三种登录模式对比

| 模式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **本地模式** | 无需配置，开箱即用 | 无真实 openid，无法云端同步 | 开发测试 |
| **云开发模式** | 官方支持，简单易用，免费额度 | 需要开通云开发 | 推荐 ⭐ |
| **自定义服务器** | 完全控制，灵活扩展 | 需要自建后端服务 | 企业应用 |

---

## 🚀 模式一：本地模式（默认）

### 特点
- ✅ 无需任何配置
- ✅ 适合快速开发测试
- ❌ 生成模拟 openid
- ❌ 无法实现真实的数据同步

### 配置方法
**无需配置！** 项目默认使用本地模式。

### 工作原理
```javascript
// app.js 中默认配置
globalData: {
  useCloudBase: false,  // 不使用云开发
  serverUrl: '',        // 不使用自定义服务器
}
```

当用户点击"微信登录"时：
1. 调用 `wx.login()` 获取 code
2. 生成随机 openid（格式：`local_时间戳_随机字符串`）
3. 保存到本地存储
4. 获取用户昵称、头像等信息

---

## ⭐ 模式二：云开发模式（推荐）

### 特点
- ✅ 官方提供，稳定可靠
- ✅ 配置简单，无需自建服务器
- ✅ 免费额度充足（每天 5 万次调用）
- ✅ 自动获取真实 openid
- ✅ 支持数据云端存储

### 配置步骤

#### 1. 开通云开发
在微信开发者工具中：
1. 点击顶部工具栏"云开发"按钮
2. 开通云开发服务（首次开通需要微信认证）
3. 创建云环境（环境名称自定义，如 `fund-prod`）
4. 记录环境 ID（后面要用）

#### 2. 上传云函数
```bash
# 在项目根目录
cd miniprogram/cloudfunctions/login

# 右键点击 login 文件夹
# 选择"上传并部署：云端安装依赖"
```

#### 3. 修改 app.js 配置
```javascript
// miniprogram/app.js
globalData: {
  useCloudBase: true,  // ✅ 启用云开发
  serverUrl: '',       
},

// 在 initCloud() 方法中填入环境 ID
initCloud() {
  wx.cloud.init({
    env: 'your-env-id',  // ✅ 填入你的云环境 ID
    traceUser: true,
  });
}
```

#### 4. 测试登录
1. 重新编译小程序
2. 点击"微信登录"
3. 检查控制台输出是否有真实 openid

### 云函数说明

**login 云函数** (`cloudfunctions/login/index.js`)
```javascript
exports.main = async (event, context) => {
  const wxContext = cloud.getWXContext()
  return {
    openid: wxContext.OPENID,      // 用户唯一标识
    appid: wxContext.APPID,        // 小程序 AppID
    unionid: wxContext.UNIONID,    // 开放平台 UnionID
    env: wxContext.ENV,            // 环境 ID
  }
}
```

### 云开发数据同步（可选扩展）

如果要实现数据云端同步，可以创建云函数：

**syncData 云函数示例：**
```javascript
// cloudfunctions/syncData/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

exports.main = async (event, context) => {
  const wxContext = cloud.getWXContext()
  const { action, data } = event

  try {
    switch (action) {
      case 'save':
        // 保存数据
        return await db.collection('user_funds').add({
          data: {
            openid: wxContext.OPENID,
            funds: data.funds,
            settings: data.settings,
            updateTime: new Date()
          }
        })

      case 'load':
        // 加载数据
        return await db.collection('user_funds')
          .where({ openid: wxContext.OPENID })
          .orderBy('updateTime', 'desc')
          .limit(1)
          .get()

      default:
        return { success: false, message: '未知操作' }
    }
  } catch (err) {
    return { success: false, error: err }
  }
}
```

## 🔧 模式三：自定义服务器

### 特点
- ✅ 完全控制
- ✅ 可集成现有系统
- ✅ 灵活扩展
- ❌ 需要自建服务器
- ❌ 需要配置域名白名单

### 配置步骤

#### 1. 后端实现登录接口

**Node.js 示例：**
```javascript
// server/api/wx-login.js
const axios = require('axios');

app.post('/api/wx-login', async (req, res) => {
  const { code } = req.body;
  
  try {
    // 换取 openid 和 session_key
    const response = await axios.get('https://api.weixin.qq.com/sns/jscode2session', {
      params: {
        appid: 'YOUR_APPID',
        secret: 'YOUR_SECRET',
        js_code: code,
        grant_type: 'authorization_code'
      }
    });

    const { openid, session_key } = response.data;

    if (openid) {
      // 生成自定义 token
      const token = generateToken(openid);

      res.json({
        success: true,
        token,
        openid,
        session_key
      });
    } else {
      res.json({
        success: false,
        message: '登录失败'
      });
    }
  } catch (error) {
    res.status(500).json({
      success: false,
      message: error.message
    });
  }
});
```

#### 2. 配置小程序域名白名单
在微信公众平台 → 开发 → 开发管理 → 服务器域名：
```
request 合法域名：
https://your-domain.com
```

#### 3. 修改 app.js 配置
```javascript
// miniprogram/app.js
globalData: {
  useCloudBase: false,
  serverUrl: 'https://your-domain.com',  // ✅ 填入你的服务器地址
}
```

## 📱 用户登录流程

### 完整流程图
```
用户点击登录
    ↓
⚠️ 直接调用 wx.getUserProfile()
（必须在点击事件的同步代码中）
    ↓
获取用户昵称、头像
    ↓
保存用户信息到本地
    ↓
调用 wx.login()
    ↓
获取临时 code
    ↓
┌─────────────────────────┐
│   根据配置选择登录模式    │
└─────────────────────────┘
    ↓
┌──────┬──────────┬──────────┐
│      │          │          │
本地   云开发    自定义服务器
模式   模式       模式
│      │          │
↓      ↓          ↓
生成   调用云函数  发送到服务器
模拟   获取openid  换取openid
openid
│      │          │
└──────┴──────────┴──────────┘
    ↓
保存 openid 到本地存储
    ↓
登录完成
```

### ⚠️ 重要：getUserProfile 调用限制

`wx.getUserProfile()` **必须**在用户点击事件中**直接调用**，不能：
- ❌ 在异步回调中调用
- ❌ 在 `async/await` 之后调用
- ❌ 在定时器中调用
- ❌ 通过 Promise 链调用

**正确做法：**
```javascript
// ✅ 正确：在点击事件中直接调用
handleWxLogin() {
  wx.getUserProfile({
    desc: '用于完善用户资料',
    success: (res) => {
      // 获取到用户信息后，再进行其他操作
      const userInfo = res.userInfo;
      // ... 保存信息
      // ... 调用 wx.login() 等其他操作
    }
  });
}

// ❌ 错误：在 async 函数中调用
async handleWxLogin() {
  await someAsyncOperation();  // 异步操作后
  wx.getUserProfile({...});    // ❌ 会报错！
}
```

### 代码调用示例
```javascript
// pages/login/login.js
handleWxLogin() {
  this.setData({ loading: true });

  // 1. 先获取用户信息（必须在点击事件同步代码中）
  wx.getUserProfile({
    desc: '用于完善用户资料',
    success: (res) => {
      const userInfo = res.userInfo;
      wx.setStorageSync('userInfo', userInfo);

      // 2. 再调用登录获取 openid
      app.wxLogin()
        .then(() => {
          wx.showToast({ title: '登录成功' });
          wx.switchTab({ url: '/pages/index/index' });
        })
        .catch(error => {
          console.error('登录失败:', error);
        });
    },
    fail: (error) => {
      console.error('用户拒绝授权', error);
    }
  });
}
```

## 🔐 安全说明

### ⚠️ 重要提醒

1. **不要在小程序中硬编码 AppSecret**
   ```javascript
   // ❌ 错误做法
   const secret = 'your-app-secret';  
   
   // ✅ 正确做法：在后端服务器中使用
   ```

2. **session_key 不要传输给客户端**
   ```javascript
   // ❌ 错误
   return { openid, session_key };
   
   // ✅ 正确
   return { openid, token };
   ```

3. **使用 HTTPS**
   - 自定义服务器必须使用 HTTPS
   - 配置有效的 SSL 证书

4. **Token 安全**
   ```javascript
   // 建议使用 JWT 生成 token
   const jwt = require('jsonwebtoken');
   const token = jwt.sign({ openid }, SECRET_KEY, { expiresIn: '7d' });
   ```

## 🐛 常见问题

### 1. 云函数调用失败
**症状：** 控制台提示 `cloud.callFunction 失败`

**解决：**
- 检查云开发是否已开通
- 检查 `env` 环境 ID 是否正确
- 检查云函数是否已上传部署
- 检查网络是否正常

### 2. 获取不到用户信息
**症状：** `wx.getUserProfile` 失败或报错 "can only be invoked by user TAP gesture"

**原因：**
- ❌ 在异步操作之后调用
- ❌ 在 `async/await` 之后调用  
- ❌ 用户拒绝授权

**解决：**
```javascript
// ❌ 错误写法
async handleLogin() {
  await app.wxLogin();           // 异步操作
  await app.getUserInfo();       // ❌ 报错！
}

// ✅ 正确写法
handleLogin() {
  wx.getUserProfile({            // ✅ 直接调用
    desc: '用于完善用户资料',
    success: (res) => {
      // 先获取用户信息
      const userInfo = res.userInfo;
      wx.setStorageSync('userInfo', userInfo);
      
      // 再进行其他操作
      app.wxLogin().then(() => {
        console.log('登录成功');
      });
    }
  });
}
```

**关键点：**
1. `wx.getUserProfile()` 必须在点击事件的**同步代码**中调用
2. 不能使用 `async/await`
3. 不能在任何异步操作之后调用
4. 必须有用户的主动点击触发

### 3. openid 为 undefined
**症状：** 登录后 openid 是 undefined

**解决：**
- **云开发模式：** 检查云函数返回值
- **自定义服务器：** 检查接口返回数据格式
- **本地模式：** 应该有 `local_` 前缀的值

### 4. request:fail url not in domain list
**症状：** 自定义服务器请求失败

**解决：**
- 在微信公众平台配置服务器域名白名单
- 开发时可以在"详情 → 本地设置"中勾选"不校验合法域名"
