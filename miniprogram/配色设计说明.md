# 基金小程序配色设计说明

## 📘 设计理念

本小程序采用专业的金融行业配色标准，旨在营造稳健、专业、科技感的视觉体验。

---

## 🎨 核心配色方案

### 1. 主色调 - 金融科技蓝

| 颜色名称 | 色值 | 用途 |
|---------|------|------|
| **核心蓝** | `#1677FF` | 主要按钮、强调元素、品牌色 |
| **深蓝** | `#0052D9` | 渐变配色、深色状态 |
| **浅蓝** | `#4096FF` | 次要元素、辅助配色 |

**应用场景：**
- 页面背景渐变（`#0052D9` → `#1677FF` → `#4096FF`）
- 主要操作按钮
- 图标高亮
- 链接文字

---

### 2. 涨跌配色 - 中国标准（红涨绿跌）

| 状态 | 颜色名称 | 色值 | 透明度版本 |
|------|---------|------|-----------|
| **涨** | 朱红 | `#D32F2F` | `rgba(211, 47, 47, 0.15)` |
| **涨（浅）** | 浅朱红 | `#EF5350` | 用于强调 |
| **跌** | 墨绿 | `#388E3C` | `rgba(56, 142, 60, 0.15)` |
| **跌（浅）** | 浅墨绿 | `#66BB6A` | 用于强调 |

**重要说明：**
- ✅ 符合中国大陆用户习惯：**红色代表上涨，绿色代表下跌**
- 🎨 使用稍显高级的"朱红"和"墨绿"，避免高饱和度造成视觉疲劳
- 📊 图表渐变填充使用 15%-30% 透明度，增强专业感

**未来扩展：**
如需支持国际/港股市场（绿涨红跌），可在设置中提供"涨跌色切换"功能。

---

### 3. 预警与提示色

| 用途 | 颜色名称 | 色值 | 背景色 |
|------|---------|------|--------|
| **风险警告** | 琥珀色 | `#FFA726` | `#FFF3E0` |
| **删除/危险** | 朱红 | `#D32F2F` | `#FFEBEE` |
| **收藏/星标** | 金色 | `#FFA726` | `#FFF3E0` |

**应用场景：**
- 高溢价风险提示
- 基金经理变动通知
- 删除确认按钮
- 警告标签（`.warning-badge`）

---

### 4. 中性色系统

| 用途 | 色值 | 说明 |
|------|------|------|
| **主要文字** | `#1A1A1A` | 标题、基金名称 |
| **次要文字** | `#666666` | 说明文字、标签 |
| **辅助文字** | `#999999` | 基金代码、时间戳 |
| **禁用文字** | `#CCCCCC` | 不可操作状态 |

---

### 5. 背景与卡片

| 元素 | 色值 | 说明 |
|------|------|------|
| **页面背景** | `#F5F5F5` | 浅灰色，增加呼吸感 |
| **卡片背景** | `#FFFFFF` | 纯白色 |
| **数据区背景** | `#FAFAFA` | 更浅的灰色 |
| **分割线** | `#E0E0E0` | 主要分割线 |
| **浅分割线** | `#F0F0F0` | 次要分割线 |

**设计原则：**
- 卡片使用纯白背景（`#FFFFFF`），与浅灰页面背景形成对比
- 卡片阴影：`0 2rpx 12rpx rgba(0, 0, 0, 0.08)`，轻盈不压抑
- 边框：`1rpx solid #F0F0F0`，增加层次感

## 🔤 字体规范

### 金融数字专用字体

```css
.finance-number {
  font-family: 'DIN Alternate', 'Roboto', 'Helvetica Neue', -apple-system, sans-serif;
  font-variant-numeric: tabular-nums; /* 等宽数字 */
  letter-spacing: 0.5rpx;
}
```

**应用场景：**
- 基金净值（`.value-number`）
- 涨跌幅（`.value-change`）
- 持仓权重（`.holding-weight`）
- 涨跌幅百分比

**为什么使用 DIN/Roboto？**
- ✅ 等宽设计，数字对齐整齐
- ✅ 垂直感强，精准专业
- ✅ 易读性好，适合金融数据展示

## 📊 图表配色详解

### 分时图渐变填充

**涨势渐变（红色系）：**
```javascript
gradient.addColorStop(0, 'rgba(211, 47, 47, 0.3)');   // 顶部 30% 透明度
gradient.addColorStop(0.5, 'rgba(211, 47, 47, 0.1)'); // 中部 10% 透明度
gradient.addColorStop(1, 'rgba(211, 47, 47, 0)');     // 底部 0% 透明度
```

**跌势渐变（绿色系）：**
```javascript
gradient.addColorStop(0, 'rgba(56, 142, 60, 0.3)');   // 顶部 30% 透明度
gradient.addColorStop(0.5, 'rgba(56, 142, 60, 0.1)'); // 中部 10% 透明度
gradient.addColorStop(1, 'rgba(56, 142, 60, 0)');     // 底部 0% 透明度
```

**效果说明：**
- 从上到下逐渐透明，形成自然的视觉层次
- 避免单线条显得单薄，增强图表的专业感
- 提升数据的可读性

## 🚀 最佳实践

### ✅ 推荐做法

1. **基金净值数字**：始终使用 `.finance-number` 类
2. **涨跌数据**：根据正负值动态添加 `.rise` 或 `.fall`
3. **卡片设计**：使用纯白背景 + 浅灰页面背景形成对比
4. **图表渐变**：使用半透明渐变填充，增强专业感
5. **风险提示**：使用琥珀色（`#FFA726`）和浅橙背景（`#FFF3E0`）

### ❌ 避免事项

1. 不要使用高饱和度的红绿色（如 `#FF0000` / `#00FF00`）
2. 不要在深色背景上使用浅灰文字（对比度不足）
3. 不要混用中国标准和国际标准的涨跌色
4. 不要在金融数字上使用非等宽字体
5. 不要过度使用阴影效果，保持轻盈感

---

## 📐 无障碍设计

- **对比度**：文字与背景对比度≥4.5:1（符合 WCAG AA 标准）
- **色盲友好**：涨跌同时使用颜色和符号（+/-）标识
- **触控区域**：按钮最小点击区域≥44rpx × 44rpx
